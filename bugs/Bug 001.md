# BUG-001 — Dashboard não exibe corretamente as Receitas do Mês

## Tipo
Bug funcional — cálculo e exibição de totais

## Descrição

O dashboard exibe R$ 0,00 nos cards ao carregar a página pela primeira vez, mesmo com dados existentes. Os valores só aparecem corretamente após uma ação que force uma nova requisição, como cadastrar uma nova transação. Isso indica que o carregamento inicial dos dados no componente não está sendo disparado corretamente.

## Passos para reproduzir

1. Acessar a tela de Transações
2. Cadastrar uma transação do tipo **Receita** com data no mês atual (ex: 01/04/2026, valor R$ 5.000,00)
3. Voltar ao Dashboard
4. Observar o card **Receitas do Mês**

## Comportamento esperado

O card **Receitas do Mês** deve exibir `R$ 5.000,00`.

## Comportamento encontrado

O card exibe `R$ 0,00` mesmo com a receita aparecendo corretamente em **Últimas Transações** logo abaixo.

## Evidência

- Dashboard: Receitas do Mês = `R$ 0,00`
- Últimas Transações (mesma tela): "Salario" em 01/04/2026 = `R$ 5.000,00`
- Relatório Totais por Pessoa: Thiago Razera — Total Receitas = `R$ 5.000,00` ✅

## Impacto

🔴 Alto — O Dashboard é a tela principal do sistema. Exibir totais incorretos compromete diretamente a confiabilidade do controle financeiro, que é o propósito central da aplicação.
