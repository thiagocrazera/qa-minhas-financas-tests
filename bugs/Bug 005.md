# BUG-005 — Testabilidade: Setters internos em `Transacao` impedem testes unitários externos

## Tipo
Problema de design / testabilidade

## Descrição

A classe `Transacao` possui as regras de negócio mais críticas do sistema aplicadas diretamente nos setters de `Pessoa` e `Categoria`. Porém, ambos os setters são definidos como `internal`:

```csharp
public Categoria? Categoria
{
    get => _categoria;
    internal set { ... }
}

public Pessoa? Pessoa
{
    get => _pessoa;
    internal set { ... }
}
```

## Problema

Por serem `internal`, esses setters só são acessíveis dentro do mesmo assembly (`MinhasFinancas.Domain`). O projeto de testes (`MinhasFinancas.Tests`) é externo e não consegue atribuir `Pessoa` ou `Categoria` diretamente em `Transacao`.

Como a regra do desafio proíbe qualquer alteração no código-fonte, não foi possível contornar essa limitação.

## Impacto

🟡 Médio — Não é um bug funcional, mas impede a cobertura de testes unitários nas regras de negócio mais importantes do sistema.

## O que seria necessário para corrigir

Adicionar no `.csproj` do projeto de domínio:

```xml
<ItemGroup>
  <InternalsVisibleTo Include="MinhasFinancas.Tests" />
</ItemGroup>
```
